# Health Assistant — LLM Evaluation Prototype

**Language:** English · [Türkçe](./README.tr.md)

A Streamlit app that **compares two LLM APIs** (Google Gemini and OpenAI GPT) on Turkish health-assistant conversations, with intent classification over 8 categories and a keyless demo mode.

> ## ⚠️ Safety notice
> **This project is an educational and evaluation prototype. It does not provide medical diagnosis, treatment, or emergency services.** Responses are generated by general-purpose language models and may be wrong. For any real health concern, consult a qualified professional; in an emergency, contact local emergency services immediately. All output requires human review.

**Status:** Educational / LLM-evaluation prototype. No fine-tuned model and no benchmark results are shipped (see below).

---

## What this project actually is

To avoid overclaiming, here is the precise nature of each component:

| Component | What it is | What it is **not** |
|---|---|---|
| Gemini / OpenAI integration | **Prompt-based inference** against hosted APIs | Not fine-tuned; no custom weights trained |
| Intent handling | 8-category intent labels; keyword-based routing in demo mode | Not a trained ML classifier (in demo mode) |
| Model comparison | Side-by-side response + latency/quality comparison harness | Not a published benchmark — results are computed live, not stored |
| Demo mode | Rule-based canned responses, no API key needed | Not a real LLM call |

No model is fine-tuned anywhere in this repository. Do not describe Gemini/OpenAI here as "fine-tuned".

## Dataset

Single labeled CSV: `data/health_assistant_dataset.csv`.

| Property | Value |
|---|---|
| Rows | 1,250 |
| Columns | `text`, `intent`, `response` |
| Intent categories | 8 |

Intent distribution (verified from the CSV):

| Intent | Count |
|---|---:|
| symptom_inquiry | 400 |
| medication_info | 200 |
| appointment_booking | 200 |
| general_health | 150 |
| emergency | 100 |
| doctor_recommendation | 100 |
| greeting | 50 |
| goodbye | 50 |

> There is **no committed train/validation/test split** and no stored evaluation output. Any precision/recall/F1 must be computed at runtime by the comparison script; those numbers are not benchmarked or frozen here.

## Emergency & responsible-AI handling

- `emergency` is an explicit intent (100 samples) so the app can surface a "seek immediate help" style message rather than attempting advice.
- **No-diagnosis policy:** the app must not present output as a diagnosis.
- **Privacy limitation:** conversations may be sent to third-party LLM APIs (Google/OpenAI) when keys are configured; do not enter real personal health information.

## Installation

```bash
git clone https://github.com/Mertcan-Gelbal/health-assistant-chatbot.git
cd health-assistant-chatbot
pip install -r requirements.txt
```

### API keys (optional — demo mode needs none)

```bash
cp .env.example .env
# edit .env and add GOOGLE_API_KEY and/or OPENAI_API_KEY
```

Never hardcode keys in source. Keys are read from the environment (see `.env.example`).

### Run

```bash
streamlit run run_app.py
```

The app opens at http://localhost:8501. Without keys it runs in **demo mode** (rule-based responses).

## Results

**No benchmark results are available yet.** The app compares models live (response, latency, intent match); it does not ship a frozen results table. When a fixed evaluation is added, report accuracy/latency with the model versions and dataset split used.

## Limitations

- LLM output can be factually wrong or unsafe for health topics (see safety notice).
- No trained classifier in demo mode; keyword routing only.
- No frozen benchmark; comparison metrics are computed on the fly.
- Third-party API calls have privacy implications.
